# AGENTS.md

## 项目目标

本项目是一个思源笔记插件，用于：

- 在思源文档中插入 TickTick 任务卡片；
- 管理七种任务状态；
- 通过任务中心动态汇总全部任务；
- 从任务中心定位、打开和修改原任务。

完整需求见：

- `docs/DEVELOPMENT_TICKET.md`

## 核心架构规则

- 原文档中的任务块属性是唯一数据源。
- 任务中心只提供动态视图，不保存任务副本。
- 状态查询必须使用稳定的状态 ID。
- 不使用 Tag 或全文搜索作为主要状态索引。
- MVP 不接入 TickTick API、OAuth 或双向同步。
- 不直接读写思源工作空间中的底层数据文件。
- 涉及块数据时优先使用思源官方 API。

## 开发规则

- 使用 TypeScript、Vite 和 Svelte。
- 先阅读现有代码和开发文档，再进行修改。
- 将修改拆分为小阶段，不一次实现整个插件。
- 每完成一个阶段后运行类型检查和生产构建。
- 保持 diff 聚焦，不实现开发文档之外的通用功能。
- 不主动修改需求文档。
- 不在未经用户明确允许时 commit、push、创建 tag 或发布 Release。
- 不直接向 `main` 推送未经测试的代码。

## 当前支持范围

MVP 优先支持：

- macOS 桌面版思源；
- 用户当前使用的思源版本；
- 默认浅色和深色主题。

移动端和其他平台不属于当前优先范围。
