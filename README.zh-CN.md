# 思源滴答任务中心

滴答任务中心是一个用于在思源文档中关联 TickTick 任务的插件。任务数据保存在普通块的结构化属性中，源块同时保留可读的 Markdown 降级内容：

```markdown
滴答任务：[任务名称](https://dida365.com/...)
```

规划中的 MVP 将提供任务卡片增强显示、七种本地任务状态以及动态任务中心页签。原文档中的任务块始终是唯一数据源，任务中心不会维护第二份任务数据库。

## 当前状态

插件目前已经支持：

- 通过 Protyle 斜杠菜单创建 TickTick 任务；
- 在源文档中保存可读的 Markdown 降级块；
- 使用七项结构化块属性持久化任务数据；
- 校验任务输入，并在属性持久化失败时回滚已插入的块；
- 使用自动测试覆盖校验、Markdown 转义、属性生成、重复提交控制和回滚行为。

任务卡片增强渲染、任务编辑和任务中心尚未实现。

## MVP 边界

- 仅接受 `dida365.com` 和 `ticktick.com` 的 HTTPS 任务链接。
- 通过思源官方 API 操作块数据。
- 不接入 TickTick API、OAuth 或双向同步。
- 不使用标签或全文搜索作为主要状态索引。
- 只构建思源前端插件，不包含 Kernel 插件。

当前主要在 macOS 桌面版思源中开发和测试，暂不承诺其他平台的兼容性支持。

## 开发

开发环境需要 Node.js 20 和 pnpm。

```bash
pnpm install
pnpm typecheck
pnpm test
pnpm build
```

生产构建会生成 `dist/` 和 `package.zip`。
