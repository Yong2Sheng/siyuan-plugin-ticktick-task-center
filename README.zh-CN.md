# 思源滴答任务中心

滴答任务中心是一个用于在思源文档中关联 TickTick 任务的插件。任务数据保存在普通块的结构化属性中，源块同时保留可读的 Markdown 降级内容：

```markdown
滴答任务：[任务名称](https://dida365.com/...)
```

插件已提供任务卡片增强显示、七种本地任务状态以及动态任务中心页签。原文档中的任务块始终是唯一数据源，任务中心不会维护第二份任务数据库。

## 当前状态

插件目前已经支持：

- 通过 Protyle 斜杠菜单创建 TickTick 任务；
- 在源文档中保存可读的 Markdown 降级块；
- 使用七项结构化块属性持久化任务数据；
- 校验任务输入，并在属性持久化失败时回滚已插入的块；
- 以任务卡片增强显示身份标签、任务链接和可交互的本地化状态徽章；
- 非破坏性保留底层 Markdown 和块属性，插件关闭后恢复普通 Markdown；
- 点击状态徽章打开完整任务编辑窗口，修改任务标题、TickTick URL 和任务状态；
- 编辑成功后同步更新七项结构化属性，并在标题或 URL 变化时同步更新 Markdown 降级内容；`created-at` 保持不变，只有实际编辑才更新 `updated-at`；
- 通过乐观并发冲突检测避免覆盖其他位置的更新，并在属性写入失败时回滚 Markdown；
- 通过顶栏按钮打开单例的“滴答任务中心”页签，并动态汇总整个工作空间内通过验证的任务；
- 使用条件聚合读取七项任务属性，使每个任务对应一条字段确定的 SQL 聚合结果；
- 默认显示活跃任务，支持活跃 / 已结束 / 全部筛选，以及任务标题、来源文档和本地化状态搜索；
- 默认按任务属性中的 `updated-at` 降序稳定排列，并显示全量、活跃和已结束任务统计；
- 支持定位思源原任务块、安全打开 TickTick 链接，并通过刷新按钮同步普通文档或其他客户端产生的变化；
- 在任务中心通过状态徽章复用现有完整编辑窗口，保存成功后直接更新当前内存列表，不等待思源 SQL 属性索引；
- 使用仅存在于当前页签生命周期内的最近编辑覆盖层，避免保存后立即手动刷新时，暂时滞后的 SQL 结果让界面回退；
- SQL 仅在任务中心首次打开、用户点击刷新或查询失败后点击重试时执行，不轮询，也不在后台自动查询；
- 区分 SQL 属性行暂时不完整与七项属性齐全但数据确实无效的任务块；
- 使用自动测试覆盖校验、Markdown 转义、属性生成、创建与编辑流程、任务卡片生命周期、任务中心聚合、筛选、刷新控制、页签生命周期和 UI 行为。

批量操作、任务删除、任务右键菜单和 TickTick API 同步尚未实现。

在普通文档中创建或编辑任务后，需要点击任务中心的“刷新”按钮同步这些外部变化。最近编辑覆盖层只是当前页签的临时视图状态，页签关闭后即清除；七项任务块属性始终是唯一数据源。

## MVP 边界

- 仅接受 `dida365.com` 和 `ticktick.com` 的 HTTPS 任务链接。
- 通过思源官方 API 操作块数据。
- 不接入 TickTick API、OAuth 或双向同步。
- 不使用标签或全文搜索作为主要状态索引。
- 只构建思源前端插件，不包含 Kernel 插件。

当前主要在 macOS 桌面版思源中开发和测试，暂不承诺其他平台的兼容性支持。

## 开发

开发环境需要 Node.js 20 和 pnpm。

```bash
pnpm install
pnpm test
pnpm run check
pnpm build
```

生产构建会生成 `dist/` 和 `package.zip`。
